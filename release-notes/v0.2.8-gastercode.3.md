# Gaster Code V 0.2.8-gastercode.3

这是一次绘图稳定性补丁。重点是把 Gaster Code 绘图请求从长连接同步等待改为 G-Master API 异步图片任务，避免复杂提示词或长耗时生成被上游网关超时截断。

## 主要变化

- 绘图页优先调用 G-Master API 的 `POST /v1/images/generations/async` 创建任务，再轮询 `/v1/images/jobs/{job_id}` 获取结果。
- 桌面端生成请求最长等待时间提升到 15 分钟，覆盖 G-Master API 队列中的长耗时图片生成。
- 服务器端会把异步任务中的上游 `504`、`524` 和 `upstream_timeout` 归类为稳定的图片生成超时错误。

## 提示词处理

- 生成图片时保留用户输入的原始提示词，不会为了规避超时自动压缩、改写或裁剪复杂提示词。
- 前端只使用 trim 判断提示词是否为空，实际提交给本地 server 和 G-Master API 的 prompt 保持原样。
- 提示词增强功能也不再预先去除用户输入前后的空白字符。

## 修复和稳定性

- 避免同步 `/v1/images/generations` 长连接在复杂提示词下等待到网关 524。
- 兼容旧服务端：如果异步接口不存在，会回退到同步图片接口。
- 图片任务轮询失败、创建失败和上游中断时，返回更明确的错误分类，避免页面只展示原始网络错误。
- 补充回归测试，覆盖异步任务创建、轮询成功、524 失败分类和原始提示词保留。

## 验证重点

- 复杂提示词生成时，请求先创建异步任务，再轮询任务结果，不再依赖同步长连接。
- 用户输入的原始提示词应原样进入生成请求。
- G-Master API 返回上游超时时，绘图页展示稳定的超时/上游错误提示，而不是原始 `Load failed`。
- `images-api` server 测试、Drawing 页面测试和 PR 质量门禁保持通过。
