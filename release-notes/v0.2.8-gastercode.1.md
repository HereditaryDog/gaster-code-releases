# Gaster Code V 0.2.8-gastercode.1

这是一次目标管理、桌面运行环境和发布稳定性更新。重点是让长任务目标更容易持续追踪，让 Agent 进度与结果更可读，并减少桌面端从 Finder、Dock、定时任务或 MCP 启动时遇到的本机环境不一致问题。

## 主要变化

- 新增 `/goal` 目标跟踪：可以设置、查看和清除当前目标，目标状态会在桌面会话中保持可见，并在任务完成后写入会话记录。
- 优化 Agent 和后台任务展示：Agent 进度保留在对应卡片内，结构化结果会优先展示为可读文本，减少原始 JSON 干扰。
- 桌面 CLI、定时任务、MCP stdio 和 MCP 预检会继承用户终端 shell 环境，改善 `PATH`、Homebrew、Node、Bun、npx 等命令在桌面启动后找不到的问题。
- 增强会话恢复和工作目录兼容：修复旧 Windows 会话路径、临时 worktree 标签和 stale workdir 恢复场景。
- 设置页新增更新代理选项：默认使用系统代理，也可以为应用更新检查和安装包下载单独设置 HTTP/HTTPS 代理。
- 改进便携模式和 Windows 终端设置，减少换机、外接盘和 Windows shell 配置带来的启动失败。
- 新增可重复启动本地 Web UI 的脚本，并强化 Windows / Tauri 打包时的 Bun 编译缓存配置。

## 修复和稳定性

- 修复自定义斜杠命令在 live 更新后不可见的问题。
- 修复多问题确认、切换标签页未发送草稿、选中文本加入聊天、后台任务完成态展示等桌面交互细节。
- 修复空模型槽位进入运行时导致 provider 配置异常的问题。
- 改进飞书 IM 首次配置引导和路径安全校验。
- 修复通知插件监听器上下文丢失问题。
- 稳定 adapter WebSocket 测试和正常关闭 session 后的状态清理。

## 安全和兼容性

- 继续保留旧配置和旧本地存储键的兼容迁移，但用户可见路径、文案和服务商流程使用 Gaster Code / G-Master API。
- 更新代理只作用于应用自身的更新检查和安装包下载，不会影响模型服务商或聊天请求。
- MCP 和 CLI 环境继承只读取用户 shell 导出的环境变量，不改变用户 shell 配置文件。

## 安装包

本版本安装包会通过桌面发布工作流构建，并同步到公开 release-only 仓库：

- macOS Apple Silicon：`Gaster-Code_0.2.8-gastercode.1_macos_arm64_dmg.dmg`
- macOS Intel：`Gaster-Code_0.2.8-gastercode.1_macos_x64_dmg.dmg`
- Windows x64：`Gaster-Code_0.2.8-gastercode.1_windows_x64_nsis.exe`
- Linux x64：`Gaster-Code_0.2.8-gastercode.1_linux_x64_deb.deb`

本次代码合并到主仓库后不会在主仓库直接构建安装包；安装包构建会在具备发布额度和签名配置的发布仓库流程中完成。

## 验证重点

- `/goal` 在 CLI 和桌面端都能正确设置、清除、显示和写入完成记录。
- Agent 卡片展示子任务进度和结构化结果时保持可读，不出现裸 JSON 刷屏。
- Finder / Dock 启动桌面端后，CLI、MCP 和定时任务仍能找到用户 shell 中配置的本机工具。
- 更新代理设置保存后，更新检查和安装流程使用对应代理，并在代理变化后重新检查更新。
- Windows x64、macOS ARM64、macOS x64、Linux x64 发布工作流保持相同版本号和安装包命名。
