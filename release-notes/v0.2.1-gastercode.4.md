# Gaster Code V 0.2.1-gastercode.4

这是一个 updater 可用性修复版本，延续 `0.2.1-gastercode.3` 的 IM adapter 品牌修复，并调整桌面端发布矩阵，避免 GitHub Actions 账单/额度限制阻断公开 release-only 仓库同步。

## 更新内容

- 修复微信配对流程中仍提示 `Claude Code` / `Claude` 的问题，现在未授权和配对成功文案都会显示 `Gaster Code`。
- 同步修复 Telegram、飞书、钉钉 adapter 的配对提示，统一通过共享 formatter 输出品牌文案。
- 将钉钉权限卡标题和普通消息标题改为 `Gaster Code`。
- 将正式发布默认矩阵调整为 macOS ARM64、macOS x64 和 Linux x64，Windows x64 与 Linux ARM64 改为手动发布时按需启用。
- 保持主仓库私有，并继续把安装包、签名文件和 `latest.json` 同步到公开的 `HereditaryDog/gaster-code-releases` release-only 仓库，供桌面端 updater 匿名访问。
- GitHub Actions 账单/额度目前阻止 runner 启动，本版本公开包先使用本地 Apple Silicon 构建产物发布，用于验证 updater 链路。

## 验证

- `bun run check:adapters`
- `bun run check:docs`
- `bun run check:native`
- `bun run quality:pr`
