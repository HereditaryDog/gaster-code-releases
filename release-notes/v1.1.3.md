# Gaster Code V 1.1.3

V 1.1.3 是 Gaster Code Electron 桌面端审计修复版本，重点改善长上下文会话切换、桌面端加载路径、发布远端卫生和 macOS 安装包体验。

## 主要更新

- 长上下文会话切换继续采用先切页面、再渐进加载内容的路径，避免点击不同对话时被大量 transcript 同步渲染阻塞。
- 活跃 transcript 增长到阈值后会自动进入渐进渲染，并在用户位于底部时保持滚动锚定。
- 桌面内容路由拆分为懒加载，Provider 设置 API 也延后加载，减少启动和页面切换时的主线程压力。
- macOS DMG 增加品牌化拖拽安装背景，安装窗口明确展示 `Gaster Code.app` 到 `Applications` 的拖拽方向。
- 正式发布矩阵收窄为 macOS Apple Silicon 和 Windows x64；macOS Intel 和 Linux 产物暂不构建。

## PR 复核

- 复核同事 PR #8 `fix: improve desktop rendering performance`。
- PR 中的 TabBar 拖拽节流、Markdown 缓存/渐进渲染、WorkspacePanel 大文件预览上限等核心性能点已经在当前 1.1.x 主线覆盖。
- 由于该 PR 分支基底较旧，本次没有直接合入旧分支，避免把 Electron 迁移后的文件和发布链路回退。

## 发布与安装包

| 平台 | 安装包 |
|------|--------|
| macOS Apple Silicon | `Gaster-Code-1.1.3-mac-arm64.dmg` |
| Windows x64 | `Gaster-Code-1.1.3-win-x64.exe` |

本地 Apple Silicon 测试包继续使用内部命名：`Gaster Code-a 1.1.3.dmg`。

macOS 用户如果首次启动提示“已损坏，无法打开”、“此文件已损坏”或“无法验证开发者”，请先将 `Gaster Code.app` 拖入 `Applications`，然后在终端执行：

```bash
xattr -cr /Applications/Gaster\ Code.app
```

如果仍然打不开，可以再执行一次带管理员权限的命令：

```bash
sudo xattr -cr /Applications/Gaster\ Code.app
```

## 验证重点

- 打开一个长上下文会话，再切到其他会话，确认页面切换先发生，长内容随后渐进出现。
- 在长会话底部继续接收流式内容，确认渲染进入渐进模式后不会把视图从底部顶走。
- 打开设置和 Provider 页面，确认懒加载路径不影响账号、余额、订阅和模型提供商配置。
- 挂载本地 macOS Apple Silicon DMG，确认安装背景、应用图标、Applications 图标和拖拽方向显示正常。
- 使用 GitHub Actions Windows x64 构建产物验证 NSIS 安装包结构。
