# Gaster Code V 0.2.0-gastercode.5

这是 Gaster Code 桌面端 updater 发布链路修复版本。它保留 `0.2.0-gastercode.4` 引入的新 updater 签名密钥，并修复正式发布时多平台构建并发创建 GitHub Release 导致资产和 `latest.json` 被拆分的问题。

## 更新内容

- 串行化桌面端 release 矩阵，避免 `tauri-action` 并发创建重复 GitHub Release。
- 确保正式发布的 `latest.json` 汇总 macOS、Linux 和 Windows updater 平台入口。
- 新增发布工作流回归测试，覆盖 release job 串行化要求、updater artifacts 开关和当前 updater 公钥。
- 更新 README、桌面端安装文档和版本入口文案到 `V 0.2.0-gastercode.5`。

## 升级说明

由于 updater 签名公钥已经在 `0.2.0-gastercode.4` 轮换，已经安装 `0.2.0-gastercode.2` / `0.2.0-gastercode.3` 的用户无法通过旧公钥验签本版本更新，需要从 GitHub Releases 手动下载并安装一次 `0.2.0-gastercode.5`。

安装本版本后，后续版本会继续使用新的 updater 签名链路，桌面端即可通过「设置 -> 关于 -> 检查更新」检测和安装更新。

## 发布验证

推送 `v0.2.0-gastercode.5` 标签后，GitHub Actions 会自动构建 macOS、Linux 和 Windows 桌面安装包，并使用本文件作为 GitHub Release 正文。

发布完成后，Release 资产中应包含 `latest.json`，并且 `latest.json` 应包含所有受支持平台的签名 updater 入口。
