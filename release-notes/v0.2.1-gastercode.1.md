# Gaster Code V 0.2.1-gastercode.1

这个版本继续完善 Gaster Code 的桌面端能力，并保留 Gaster 的产品身份、G-Master API、GPT Image 2 绘图、蓝色主题和独立 updater 链路。

## 主要更新

- 新增微信和钉钉 IM adapter，补齐 Telegram、飞书、微信、钉钉四个平台的配置入口、配对管理、默认项目和移动端权限审批。
- 支持每个平台独立 sidecar、stale sidecar 清理、IM credential 权限收紧和权限审批短回复能力。
- 新增桌面系统通知能力，覆盖权限请求、定时任务完成和长任务最终回复，并在 macOS 上使用原生 UserNotifications 桥接。
- 改进上下文用量显示，覆盖 `/context`、桌面 composer、断开会话 transcript fallback、模型上下文窗口和空会话初始状态。
- 增强诊断页：最近错误、复制错误摘要、打开日志目录、清理日志和导出脱敏诊断包。
- 修复终端 cwd，终端页和提升为独立 tab 的终端会保留当前工作目录。
- 保留 G-Master API 官方服务商、登录状态同步、模型选择排序和 GPT Image 2 绘图工作区。
- 保留 Gaster Code 蓝色主题、G 字 logo、桌面名称、bundle identifier、发布产物命名和 updater endpoint。

## 修复

- 修复桌面通知标题残留旧名称的问题，统一为 `Gaster Code`。
- 修复版本入口、README、安装文档和 release notes 指向旧 `0.2.0-gastercode.7` 的问题。
- 修复 provider 设置页在 G-Master 官方服务商与 auth strategy / context window 设置之间的冲突。

## 安装包命名

- macOS Apple Silicon：`Gaster-Code_0.2.1-gastercode.1_macos_arm64_dmg.dmg`
- macOS Intel：`Gaster-Code_0.2.1-gastercode.1_macos_x64_dmg.dmg`
- Windows x64：`Gaster-Code_0.2.1-gastercode.1_windows_x64_nsis.exe`

本版本的发布说明只记录 Gaster Code 自身能力和用户可见变化。
