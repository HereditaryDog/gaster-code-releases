# Gaster Code V 1.1.5

V 1.1.5 是 Gaster Code Electron 桌面端前端打磨版本，重点移植并验证同事公开仓库 PR #12 的桌面壳层、侧边栏、输入框附件态和上下文用量指示器修复。

## 主要更新

- 桌面壳层和侧边栏加入新的 Gaster 品牌标识、深色材质选中态和桌面窗口圆角适配。
- 聊天输入框改为更紧凑的浮动材质样式，并为图片附件预览新增独立 attachment-stage 展示态。
- 修复 `ContextUsageIndicator` 在紧凑模式和移动视口切换时可能触发的 React hook 顺序崩溃。
- 设置页和空会话页继续沿用 V 1.1.4 的分组导航、设置搜索和账号优先入口，并补充相关回归测试。

## PR 审查

- 审查并移植同事公开 release 仓库 PR #12 `fix: polish v1.1.4 desktop frontend`。
- 该 PR 原本提交到 `HereditaryDog/gaster-code-releases` release-only 仓库，公共仓库 CI 因缺少源码仓库依赖安装而在 `change-policy` 阶段失败；本次将有效源码改动移植回 `HereditaryDog/gaster-code` 主仓库并使用源码仓库门禁验证。
- 保留 V 1.1.3/V 1.1.4 的长上下文会话切换和绘制隔离保护，避免回退到旧 Electron 迁移基线。

## 发布与安装包

| 平台 | 安装包 |
|------|--------|
| macOS Apple Silicon | `Gaster-Code-1.1.5-mac-arm64.dmg` |
| Windows x64 | `Gaster-Code-1.1.5-win-x64.exe` |

本地 Apple Silicon 测试包继续使用内部命名：`Gaster Code-a 1.1.5.dmg`。

macOS 用户如果首次启动提示“已损坏，无法打开”、“此文件已损坏”或“无法验证开发者”，请先将 `Gaster Code.app` 拖入 `Applications`，然后在终端执行：

```bash
xattr -cr /Applications/Gaster\ Code.app
```

如果仍然打不开，可以再执行一次带管理员权限的命令：

```bash
sudo xattr -cr /Applications/Gaster\ Code.app
```

## 验证重点

- 打开空会话页，确认输入框默认为浮动紧凑材质样式。
- 上传图片附件，确认附件预览进入独立 attachment-stage 面板，图片缩略图和移除按钮位置正常。
- 打开长上下文会话，确认上下文用量指示器不会因紧凑/移动视口判断触发 hook 顺序错误。
- 在桌面端切换会话、设置页和侧边栏，确认主侧栏状态、选中态和窗口圆角正常。
- 使用本地 macOS Apple Silicon DMG 和 GitHub Actions Windows x64 构建产物分别验证安装包结构。
