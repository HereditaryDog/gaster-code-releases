# Gaster Code V 0.2.8-gastercode.2

这是一次桌面端会话切换性能补丁。重点是减少用户在侧边栏点击切换历史会话时的重复请求和不必要刷新，让多会话工作流更顺滑。

## 主要变化

- 会话任务列表请求增加同会话 in-flight 复用，避免侧边栏和当前会话页面在切换时同时拉取同一份任务数据。
- 历史会话的变更卡片增加短期缓存和请求复用，切回同一个已完成会话时不再重复请求 checkpoints 和工作区状态。
- 保留会话切换时清空旧任务列表的行为，避免新会话短暂显示上一个会话的任务状态。

## 修复和稳定性

- 降低多标签、多历史会话频繁切换时的前端状态抖动。
- 减少后端 `/api/cli-tasks`、turn checkpoints 和 workspace status 的重复压力。
- 补充回归测试，覆盖同会话任务请求复用和历史会话变更卡片缓存复用。

## 安装包

本版本安装包会通过桌面发布工作流构建，并同步到公开 release-only 仓库：

- macOS Apple Silicon：`Gaster-Code_0.2.8-gastercode.2_macos_arm64_dmg.dmg`

本次代码合并到主仓库后不会在主仓库直接构建安装包；安装包构建会在具备发布额度和签名配置的发布仓库流程中完成。

## 验证重点

- 在侧边栏快速切换两个历史会话时，当前会话内容能正常显示，不出现浏览器 console error。
- 切回同一个历史会话时，不重复请求同一批 turn change cards。
- 同一个 session 的任务列表请求在并发触发时只发送一次。
- `check:desktop`、`check:native` 和 PR 质量门禁保持通过。
