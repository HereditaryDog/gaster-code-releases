# Gaster Code V 0.2.6-gastercode.1

这是一次桌面端体验和本地服务安全更新。重点是让 H5 访问、会话整理、文件引用、主题、侧边栏导航和兼容模型返回值在真实使用中更稳定，同时继续保留 Gaster Code 品牌、G-Master API 服务商流程和用户自行安装的 Skills / Agents 可见性。

## 主要变化

- H5 访问改为明确开启后可用：设置页提供独立入口、Token/二维码、允许来源和公开访问地址配置；远程浏览器请求需要 H5 Token，本机桌面 WebView 和本地 SDK/adapter 调用仍保持正常启动。
- 会话列表新增批量整理能力：支持进入批量模式、选择当前可见会话、按范围选择、统一确认后批量删除，并保留逐会话清理结果。
- `@` 文件引用更贴近项目结构：优先使用 Git tracked/untracked 文件，遵守 ignore 规则，支持目录候选和非 Git 目录的搜索 fallback。
- 聊天输入草稿按会话隔离，切换标签时不会把未发送文本或附件泄漏到其他会话。
- 长 URL、Markdown 链接和用户消息内容在聊天气泡内安全换行，避免撑破消息列。
- OpenAI Responses 兼容通道支持 `prompt_tokens` / `completion_tokens` usage 字段，避免兼容服务只返回 OpenAI 风格用量时触发空值错误。
- 设置中新增纯白主题，保持 Gaster Code 蓝色品牌强调色，不引入旧项目色板。
- 侧边栏支持桌面端拖拽调整宽度，并优化字体、字重、折叠按钮和会话列表层级，长时间开发时更容易扫描。
- Skills、Agents 页面继续展示内置、项目、用户和外部安装区域，便于用户确认自行安装的扩展是否被发现。

## 安全和兼容性

- H5 默认关闭。开启后，远程 REST、WebSocket、proxy 和设置相关请求都需要正确的 H5 Token。
- 本机 `localhost` / `127.0.0.1` 桌面控制请求不会被 H5 Token 误拦截，避免桌面端启动后误判为未授权。
- 保留旧配置和旧本地存储键的兼容迁移，但用户可见路径和文案继续使用 Gaster Code / G-Master API。
