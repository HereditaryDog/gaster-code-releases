# Gaster Code V 0.2.7-gastercode.1

这是一次桌面端开发体验、扩展发现和远程适配器稳定性更新。重点是让用户自行安装的插件、Skills、Agents 更容易被刷新发现，让项目记忆和项目打开入口进入桌面端主流程，同时提升聊天输入、Markdown 展示、桌面启动和 IM 卡片流式输出的可靠性。

## 主要变化

- 插件、Skills、Agents 页面支持刷新后重新扫描用户、项目和外部安装区域，减少安装后必须重启才能看见的情况。
- 新增 Project Memory 设置页，可以查看、编辑和预览项目记忆文件，并在聊天中展示记忆保存事件。
- 侧边栏新增打开项目入口，会根据本机可用目标打开当前项目到 VS Code、Cursor 或文件管理器。
- 优化聊天输入、附件拖拽、键盘快捷键和 Markdown 渲染，降低长内容、链接、代码块和复杂排版撑破布局的概率。
- 改进桌面启动与设置缓存恢复，增加应用缩放设置，减少旧配置或异常缓存影响启动体验。
- 强化飞书和钉钉 IM 卡片流式输出：卡片接口挂起时会超时退出，并尽量完成最终内容收尾，避免消息长期卡在生成中。
- 恢复 DeepSeek 默认服务商的禁用 thinking 参数，保持模型调用形态与桌面端运行时一致。

## 安全和兼容性

- 继续保留旧配置和旧本地存储键的兼容迁移，但用户可见路径、文案和服务商流程使用 Gaster Code / G-Master API。
- 插件和扩展刷新只会重新扫描受支持的安装区域，不会删除用户自行安装的内容。
- Project Memory 文件写入限制在对应项目记忆目录内，并限制可编辑文件大小，避免路径穿越和异常大文件影响桌面端。

## 安装包

本版本安装包会通过桌面发布工作流构建，并同步到公开 release-only 仓库：

- macOS Apple Silicon：`Gaster-Code_0.2.7-gastercode.1_macos_arm64_dmg.dmg`
- macOS Intel：`Gaster-Code_0.2.7-gastercode.1_macos_x64_dmg.dmg`
- Windows x64：`Gaster-Code_0.2.7-gastercode.1_windows_x64_nsis.exe`
- Linux x64：`Gaster-Code_0.2.7-gastercode.1_linux_x64_deb.deb`

本次代码合并到主仓库后不会在主仓库直接构建安装包；安装包构建会在具备发布额度和签名配置的发布仓库流程中完成。

## 验证重点

- 插件、Skills、Agents 刷新后能发现用户、项目和外部安装区域。
- Project Memory 能列出项目、打开 Markdown 文件、编辑保存并正确预览。
- 侧边栏打开项目入口能显示可用 IDE / 文件管理器，并拒绝无效路径。
- 聊天输入、附件、Markdown、侧边栏、设置页和桌面启动检查应通过。
- 飞书、钉钉 IM 卡片在接口超时或中间帧失败时仍能尽量完成最终内容收尾。
