# Gaster Code V 1.1.2

V 1.1.2 是 Gaster Code Electron 桌面端修复版本，重点修复 GPT Image 2 绘图页的反馈状态，并临时收窄正式发布构建矩阵以节省 GitHub Actions 额度。

## 修复

- 新一轮图片生成开始时会立即清空旧预览，避免新请求失败或超时后继续显示上一张成功图片。
- Generate 和 Enhance prompt 按钮在运行中使用统一的主题色加载反馈，进度图标旋转和文字高光动画分离，并尊重 reduced-motion 设置。
- 吸收公开 PR #6 的绘图反馈修复；公开 PR #7/#8 对应的 Electron 1.1.x 前端同步内容已经在 1.1.1 线中覆盖。

## 发布与安装包

本版本为了节省 GitHub Actions 额度，正式发布工作流只构建 macOS 和 Windows 安装包：

| 平台 | 安装包 |
|------|--------|
| macOS Apple Silicon | `Gaster-Code-1.1.2-mac-arm64.dmg` |
| macOS Intel | `Gaster-Code-1.1.2-mac-x64.dmg` |
| Windows x64 | `Gaster-Code-1.1.2-win-x64.exe` |

macOS 用户如果首次启动提示“已损坏，无法打开”、“此文件已损坏”或“无法验证开发者”，请先将 `Gaster Code.app` 拖入 `Applications`，然后在终端执行：

```bash
xattr -cr /Applications/Gaster\ Code.app
```

如果仍然打不开，可以再执行一次带管理员权限的命令：

```bash
sudo xattr -cr /Applications/Gaster\ Code.app
```

本地 Apple Silicon 测试包继续使用内部命名：`Gaster Code-a 1.1.2.dmg`。

## 验证重点

- 从历史图片恢复旧预览后，发起一轮会失败或 524 超时的新生成，请确认预览区域显示空状态，而不是旧图和旧 revised prompt。
- 确认 Generate 和 Enhance prompt 运行中按钮都有主题色加载态。
- 确认 `release-desktop.yml` 的正式矩阵只包含 macOS ARM64、macOS x64 和 Windows x64。
