# Gaster Code v1.0.2

Gaster Code 1.0.2 focuses on desktop conversation switching performance, especially for sessions with long transcripts.

## Highlights

- Switches the active conversation view before hydrating the full transcript, so the app responds to the user's session click immediately.
- Defers long transcript mounting behind a lightweight loading state instead of doing the entire render in the same interaction frame.
- Uses smaller progressive transcript hydration batches in the native Tauri runtime, reducing layout pressure in macOS WKWebView.
- Keeps the browser/H5 rendering path on a larger batch size because Chromium already handles the same transcript workload smoothly.
- Avoids starting the historical session WebSocket connection synchronously from the sidebar click handler.

## Upgrade Notes

- The desktop app, Tauri metadata, Rust package metadata, root package metadata, and About page display version are all aligned to `1.0.2`.
- No user data migration is required.
- Local Apple Silicon test packages may use the `Gaster Code-a <version>.dmg` naming convention, while official release assets continue to use the GitHub Actions asset naming format.
