# Gaster Code V 1.1.0

## Highlight

V 1.1.0 是 Gaster Code 桌面端运行时升级版本，核心是从 Tauri 2 迁移到 Electron，以 Chromium 统一 macOS、Windows 和 Linux 的桌面渲染、内置浏览器、Workbench、终端、通知、托盘、菜单和窗口行为。

## 主要更新

- 桌面端底层切换到 Electron，降低系统 WebView 差异导致的跨平台行为不一致。
- 内置浏览器、Workbench 预览、截图、网页元素选择和本地文件打开链路迁移到新的 Electron host。
- 终端运行时切换到 Electron main process 管理的 PTY 链路，保留当前 shell 设置和窗口内终端体验。
- 保留 G-Master API 登录、账号中心、充值订阅、官方默认服务商和 GPT Image 2 异步绘图能力。
- 发布链路切换到 Electron Builder、electron-updater metadata 和 package-smoke。
- 发布工作流会生成 macOS dmg/zip、Windows NSIS exe、Linux AppImage/deb，并同步到公开 release-only 仓库。

## 问题修复

- 修复 Tauri/WebView 在长上下文、内置浏览器和跨平台窗口交互上的稳定性风险。
- 合并上游 v0.4.0 中与 agent slash、会话标题、图片输入、provider context、桌面重连和 sidecar 退出相关的修复。
- 修复 Electron 迁移后可能出现的本地文件打开、外链跳转、代理、全屏、窗口状态和 sidecar 进程管理问题。
- 修复图片读取在高分辨率截图场景下没有按视觉 token 预算降采样的问题。

## 安装说明

本版本从 Tauri 更新为 Electron，1.0.x 用户请手动下载并覆盖安装一次。覆盖安装不会删除本地配置、G-Master 登录状态、会话记录或自定义服务商配置。

正式安装包会发布到公开 release-only 仓库：

| 平台 | 安装包 |
| --- | --- |
| macOS Apple Silicon | `Gaster-Code-1.1.0-mac-arm64.dmg` |
| macOS Intel | `Gaster-Code-1.1.0-mac-x64.dmg` |
| Windows x64 | `Gaster-Code-1.1.0-win-x64.exe` |
| Linux x64 AppImage | `Gaster-Code-1.1.0-linux-x86_64.AppImage` |
| Linux x64 deb | `Gaster-Code-1.1.0-linux-amd64.deb` |
| Linux ARM64 AppImage | `Gaster-Code-1.1.0-linux-arm64.AppImage` |
| Linux ARM64 deb | `Gaster-Code-1.1.0-linux-arm64.deb` |

macOS 用户如果首次启动提示“已损坏，无法打开”、“此文件已损坏”或“无法验证开发者”，请先将 `Gaster Code.app` 拖入 `Applications`，然后在终端执行：

```bash
xattr -cr /Applications/Gaster\ Code.app
```

如果仍然打不开，可以再执行一次带管理员权限的命令：

```bash
sudo xattr -cr /Applications/Gaster\ Code.app
```

本地 Apple Silicon 测试包继续使用内部命名：`Gaster Code-a 1.1.0.dmg`。

## 验证重点

- macOS Apple Silicon 本地测试包：`Gaster Code-a 1.1.0.dmg`。
- 确认 G-Master API 登录、注册、余额、充值、订阅、取消订阅、恢复订阅和交易记录正常。
- 确认 GPT Image 2 绘图仍走 G-Master API 异步任务，不改写用户提示词。
- 确认内置浏览器、Workbench 截图、终端、通知、托盘、菜单、窗口隐藏恢复和 H5 LAN 访问在 Electron 包内可用。
