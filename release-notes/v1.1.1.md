# Gaster Code V 1.1.1

## Highlight

V 1.1.1 是 Gaster Code Electron 桌面端启动修复版本，修复 V 1.1.0 安装包在 macOS 等 `file://` 打包环境下可能无法加载前端资源、最终显示“Gaster Code 启动失败”的问题。

## 主要更新

- 桌面端生产构建改为生成相对 `./assets/...` 资源路径，确保 Electron 从 `app.asar/dist/index.html` 加载时可以正确读取 JS 和 CSS。
- package-smoke 增加 renderer 资源路径检查，打包产物中如果出现会破坏 `file://` 启动的 `/assets/...` 绝对路径会直接失败。
- 保留 V 1.1.0 的 Electron runtime、G-Master API 账号中心、GPT Image 2 异步绘图、Workbench、终端、通知、托盘、菜单和窗口行为。

## 问题修复

- 修复 macOS Intel 和其他正式安装包启动后 React 未挂载、8 秒后显示“Gaster Code 启动失败”的问题。
- 修复 Vite 默认绝对资源路径在 Electron `file://` 加载模式下指向系统根目录 `/assets` 的问题。
- 修复发布质量门禁没有捕获 packaged renderer 资源路径错误的问题。

## 安装说明

V 1.1.0 用户如果启动时已经看到“Gaster Code 启动失败”，请直接下载并覆盖安装 V 1.1.1。覆盖安装不会删除本地配置、G-Master 登录状态、会话记录或自定义服务商配置。

正式安装包会发布到公开 release-only 仓库：

| 平台 | 安装包 |
| --- | --- |
| macOS Apple Silicon | `Gaster-Code-1.1.1-mac-arm64.dmg` |
| macOS Intel | `Gaster-Code-1.1.1-mac-x64.dmg` |
| Windows x64 | `Gaster-Code-1.1.1-win-x64.exe` |
| Linux x64 AppImage | `Gaster-Code-1.1.1-linux-x86_64.AppImage` |
| Linux x64 deb | `Gaster-Code-1.1.1-linux-amd64.deb` |
| Linux ARM64 AppImage | `Gaster-Code-1.1.1-linux-arm64.AppImage` |
| Linux ARM64 deb | `Gaster-Code-1.1.1-linux-arm64.deb` |

macOS 用户如果首次启动提示“已损坏，无法打开”、“此文件已损坏”或“无法验证开发者”，请先将 `Gaster Code.app` 拖入 `Applications`，然后在终端执行：

```bash
xattr -cr /Applications/Gaster\ Code.app
```

如果仍然打不开，可以再执行一次带管理员权限的命令：

```bash
sudo xattr -cr /Applications/Gaster\ Code.app
```

本地 Apple Silicon 测试包继续使用内部命名：`Gaster Code-a 1.1.1.dmg`。

## 验证重点

- 确认 macOS Apple Silicon 和 macOS Intel 安装包启动后不再显示“Gaster Code 启动失败”。
- 确认打包后的 `index.html` 中启动脚本和样式使用 `./assets/...` 相对路径。
- 确认 package-smoke 可以拦截 `/assets/...` 绝对资源路径。
