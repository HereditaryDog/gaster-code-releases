# Gaster Code V 0.2.1-gastercode.5

这是一个品牌残留清理和兼容迁移版本，重点把用户可见界面、OAuth 成功页、诊断路径、服务商预设和本地存储命名统一到 Gaster Code。

## 更新内容

- 将 OAuth 成功页、恢复 CLI、桌面端设置和诊断相关文案统一为 `Gaster Code`。
- 新增 `gaster-code` CLI 入口；`gaster-code` 继续作为旧命令兼容包装器保留。
- 默认配置和诊断日志目录迁移到 `~/.claude/gaster-code`，并兼容读取旧版配置。
- 环境变量优先使用 `GASTER_CODE_*`，同时保留旧版变量作为兼容回退。
- 桌面端 localStorage 主 key 迁移到 `gaster-code-*`，首次读取旧 key 后会写入新 key 并清理旧 key。
- 删除内置服务商里的 `接口AI`、`胜算云` 预设和相关赞助展示资产。
- 清理文档、构建脚本、E2E 脚本里的非兼容用途旧项目命名。
- 修复质量门服务商枚举脚本在默认配置目录解析时的运行错误。

## 验证

- `bun run check:policy`
- `bun run check:server`
- `bun run check:desktop`
- `bun run check:native`
- `bun run check:docs`
- `bun run quality:providers`
- `bun test scripts/quality-gate/providerTargets.test.ts`
- `bun run quality:pr`：代码测试通过，但 PR 质量门按策略阻断 CLI core 改动，需要维护者审批或 `allow-cli-core-change` 标签。
- `bun run quality:gate --mode baseline --allow-live --provider-model g-master-api:main:g-master-api-main`：本地启动后长时间无输出，已手动终止；未作为通过项声明。
