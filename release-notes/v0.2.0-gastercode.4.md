# Gaster Code V 0.2.0-gastercode.4

这是 Gaster Code 桌面端 updater 签名修复发布版本。由于此前仓库缺少与旧 `pubkey` 匹配的 Tauri updater 私钥，本版本轮换为新的 updater 公钥，并恢复 CI 中的签名更新产物生成。

## 主要变化

- 轮换桌面端 Tauri updater 公钥，并把匹配的签名私钥配置到 GitHub Actions Secrets。
- 保持正式发布时生成 `latest.json` 和签名 updater artifacts，修复「设置 -> 关于 -> 检查更新」依赖的发布 manifest。
- 新增发布配置回归测试，覆盖 release CI updater artifacts 开关和当前 updater 公钥。
- 更新 README、桌面端安装文档和版本入口文案到 `V 0.2.0-gastercode.4`。

## 安装与更新说明

由于本版本更换了 updater 签名公钥，已经安装 `0.2.0-gastercode.2` / `0.2.0-gastercode.3` 的用户无法通过旧公钥验签本版本更新，需要从 GitHub Releases 手动下载并安装一次 `0.2.0-gastercode.4`。

安装本版本后，后续版本会继续使用新的 updater 签名链路，桌面端即可通过「设置 -> 关于 -> 检查更新」检测和安装更新。

## 安装包

推送 `v0.2.0-gastercode.4` 标签后，GitHub Actions 会自动构建 macOS、Linux 和 Windows 桌面安装包，并使用本文件作为 GitHub Release 正文。

发布完成后，Release 资产中应包含 `latest.json`。
