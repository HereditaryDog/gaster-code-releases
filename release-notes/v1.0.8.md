# Gaster Code V 1.0.8

## Highlight

V 1.0.8 收口本轮项目审计修复：根项目、桌面端和适配器依赖审计均已清理，桌面 Markdown 重型渲染器改为按需加载，可信 H5 启动 Token 不再出现在新链接的 query string 中，并把桌面端版本推进到 `1.0.8`。

## 主要更新

- 修复根项目、桌面端和适配器依赖审计发现的问题，同步更新 Bun、npm 和 Cargo 相关锁文件。
- 将代码块查看器与 Mermaid 渲染器改为真实懒加载，减少桌面首屏包体，并降低长上下文会话切换时的主线程压力。
- 可信 H5 手机访问的新启动链接将 Token 放入 URL hash，减少日志、代理和历史记录中的敏感 Token 暴露面。
- 继续保留 G-Master API 账号中心、官方默认服务商同步、充值、订阅管理和 GPT Image 2 异步绘图路径。

## 问题修复

- 修复桌面端依赖升级后部分测试断言与新版 Vite/Vitest 行为不一致的问题。
- 修复 Markdown 重型渲染组件在默认路径中提前进入主 bundle 的问题。
- 修复 H5 启动 Token 在 URL query 中可见的问题；旧 query 链接仍保持兼容。
- 补充依赖审计、懒加载渲染和 H5 Token 传递相关回归验证。

## 安装说明

本版本安装包会通过桌面发布工作流构建，并同步到公开 release-only 仓库：

- macOS Apple Silicon：`Gaster-Code_1.0.8_macos_arm64_dmg.dmg`
- macOS Intel：`Gaster-Code_1.0.8_macos_x64_dmg.dmg`
- Windows x64：`Gaster-Code_1.0.8_windows_x64_nsis.exe`
- Linux x64：`Gaster-Code_1.0.8_linux_x64_deb.deb`

本地 Mac 测试包仍沿用内部命名：`Gaster Code-a 1.0.8.dmg`。

## 验证重点

- 根项目、桌面端和适配器的依赖审计应无已知漏洞。
- 长上下文会话切换时，页面应先切换，再按需加载会话内容和 Markdown 重型渲染器。
- 可信 H5 新启动链接中的 Token 应在 hash 中传递；旧 query 链接仍可兼容进入。
- README、安装指南、release notes、桌面版本号和 Tauri 打包版本应保持 `1.0.8` 一致。
